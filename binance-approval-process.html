Binance money movement — approval process

Trustee Operations Cache RE Reference guide

Scope
Investor distributions
Investor redemptions
Early redemptions
Post-lock-in redemptions
Key rules
Every redemption request must have correct approval evidence before the Binance transaction can be approved.
There is only a 15-minute window from when the IM lodges the Binance request to when Cache completes both required approvals. Two approvers must be available within this window.
Early redemptions require an RE-approved ticket. Post-lock-in redemptions require matching Admin Portal transactions.
Process steps
1
IM submits request to Cache RE Investment Manager

The Investment Manager raises a ticket to Cache RE requesting Binance approval. The request must be complete before Cache RE can begin validation.

The ticket must include:

  • The agreed day and time for the Binance transaction to be processed
  • The relevant transaction ID(s) for the full amount Cache will need to approve
  • The RE-approved ticket (early redemptions only)
  • The total amount expected to be approved in Binance
Both parties must agree on the exact day and time before Cache RE proceeds.
2
Cache RE validates the request Cache RE

Before the agreed processing time, Cache RE performs pre-approval checks. All items must be verified before proceeding.

Cache RE must confirm:

  • Admin Portal transaction(s) exist for the full redemption amount (post-lock-in)
  • The RE approval ticket exists and is properly approved (early redemptions)
  • The amount requested aligns with the supporting evidence
  • Current USDT/AUD exchange rate checked online — any variance is reasonable
  • Full Binance approval amount recorded on the ticket
3
Create a processing reminder Cache RE

Once the time is agreed, Cache RE sets a reminder in Slack or Teams so approvers are ready at the right moment.

The reminder must include:

  • The ticket reference
  • The agreed approval time
  • The expected approval amount
  • A note that two Cache approvers are required within the 15-minute window
4
Binance request received — pre-approval checks Both approvers

When the Binance request comes through, both approvers independently verify the request before approving.

Both approvers must verify:

  • The Binance approval request matches the agreed ticket
  • The total amount in Binance corresponds to the amount recorded on the ticket
  • Any difference is explainable by the USDT/AUD exchange rate
  • The request is being approved within the agreed 15-minute window
Do not approve if the amount cannot be reconciled or if you are outside the agreed window.
5
Approve the Binance transaction Both approvers

Once all checks in step 4 are complete and passed, both Cache approvers approve the transaction in Binance.

6
Confirm receipt with FundOps Cache RE

After approval, the approver contacts FundOps to confirm the funds have been received in Zerocap.

The ticket must not be closed until FundOps has confirmed receipt in Zerocap.
7
Record and close the ticket Cache RE

Cache RE updates the ticket with a full record of the approval before closing.

The ticket record must include:

  • Confirmation that the Binance request was approved
  • Confirmation that the amount matched the approved ticket or Admin Portal evidence
  • Exchange rate variance noted (if applicable)
  • FundOps confirmation of receipt in Zerocap
  • Date and time of approval
  • Names of both approvers
Do not approve if any of the following apply
No approved RE ticket for an early redemption
No matching Admin Portal transactions for a post-lock-in redemption
Amount does not match the amount on the ticket
Exchange rate variance cannot be explained
Request is outside the agreed processing window
Two Cache approvers not available within the 15-minute window
FundOps cannot confirm receipt in Zerocap — do not close the ticket
